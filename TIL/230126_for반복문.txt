# 좋은 관습 : 반복적인 셀렉터는 변수에 넣어서 쓰기

셀렉터 하나 쓸 때 마다 html을 쭉 읽고 찾아야하기 때문에
셀렉터 문법은 기본적으로 작동시간이 오래걸린다.
html이 길고 복잡할 수록 더 오래걸린다.
따라서 반복적으로 등장하면 변수에 넣어서 쓰는게 좋다.

# 코드 복붙하고 싶으면 for 반복문
for (횟수){
  복붙할 코드
}

# 확장성있는 코드로 바꾸기
1. 원하는 기능이 잘 구현되었는가
2. 확장성좋은가
3. 나중에 관리가 쉬울 것인가
4. 성능문제 없는가

# 오늘의 결론 : 
for 문법의 용도를 잘 기억해두자.
비슷한 코드들을 직접 복붙하는게 귀찮으면 for 반복문을 쓰는 것일 뿐이지,
for 부터 써놓고 무슨 코드를 채울지 고민하면 안된다!
